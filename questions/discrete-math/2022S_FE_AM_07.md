# 2022S_FE_AM_07

What is the worst-case time complexity of binary search on a sorted array
of `n` elements?

a) O(1)
b) O(log n)
c) O(n)
d) O(n log n)
?
b) O(log n)

### Explanation

Each comparison halves the search range, so at most `⌈log₂ n⌉` comparisons
are needed before the range shrinks to zero. That gives a worst-case
complexity of **O(log n)**.

### Why the Others Won't Fit

**a) O(1)**:

- Only the best case (target is the first element inspected) is constant.

**c) O(n)**:

- That's linear search over an unsorted array, not binary search.

**d) O(n log n)**:

- Typical for comparison-based sorts (merge sort, heap sort), not for search.
