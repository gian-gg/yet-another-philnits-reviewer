# 2021A_FE_AM_45

Which SQL clause filters rows **after** aggregation has been applied?

a) WHERE
b) GROUP BY
c) HAVING
d) ORDER BY
?
c) HAVING

### Explanation

`HAVING` filters the result of an aggregation — it evaluates against the
groups produced by `GROUP BY`. Logical evaluation order of a `SELECT` is:
`FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY`.

### Why the Others Won't Fit

**a) WHERE**:

- Filters rows _before_ grouping/aggregation. It cannot reference aggregate
  results like `SUM(x) > 10`.

**b) GROUP BY**:

- Defines the grouping; it does not itself filter.

**d) ORDER BY**:

- Sorts the final result set and runs last.
